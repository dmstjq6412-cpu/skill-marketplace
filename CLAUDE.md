# CLAUDE.md — Skill Marketplace

## 필수 규칙: git 작업 시 브랜치 확인 (git-guard-claude)

**코드 작업(파일 편집, 기능 구현, 버그 수정)을 시작하기 전, 반드시 현재 브랜치가 올바른지 확인할 것.**

- `main` / `master` / `develop` 브랜치에서 직접 코드를 수정하지 말 것
- 적절한 feature 브랜치가 없으면 git-guard-claude 스킬을 호출해 브랜치를 먼저 생성

> 이유: 잘못된 브랜치에서 작업하면 컨벤션 위반 + 머지 충돌 위험이 생긴다.

---

## [강제] 코드 작업 전 tdd-guard-claude 실행 — 예외 없음

**⛔ 코드 파일을 작성하거나 수정하기 전에 반드시 `Skill(tdd-guard-claude)`를 실행해야 한다. 이 단계를 건너뛰는 것은 허용되지 않는다.**

적용 대상 (이 중 하나라도 해당하면 무조건 실행):
- 새 함수, 컴포넌트, API 엔드포인트 생성
- 기존 로직 수정 또는 리팩토링
- 버그 수정

예외 — **오직 아래 경우만 면제**:
- CSS/Tailwind 클래스 변경만 있는 경우
- 설정 파일 (vite.config.js, .eslintrc 등)
- package.json 의존성 업데이트

> **위반 시**: 테스트 없이 작성된 코드는 즉시 중단하고, tdd-guard-claude를 실행한 뒤 다시 작업한다.
> "필요 없을 것 같다", "작은 변경이다" 같은 판단으로 건너뛰는 것은 허용되지 않는다.
