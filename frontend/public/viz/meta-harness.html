Meta Harness — 스킬 역할 & 인터랙션

코어 하네스를 관찰하고 개선 판단 근거를 쌓는 inspecting tool 레이어

1 두 레이어 구조
코어 하네스 — 실행 레이어
🧪 tdd-guard-claude  코드 작성 시 테스트 강제
🔒 git-guard-claude  브랜치/커밋/PR 컨벤션 + 리뷰 파이프라인
🛡 security-guard  OWASP Top 10 보안 취약점 탐지

→ 매 코드 작업 시 자동 발동. 직접 품질을 통제.

메타 하네스 — 관찰 레이어
📊 harness-analysis  토큰·REJECT율 수집
📝 harness-log  일지·블루프린트·regression_signal
📚 harness-reference  아티클 저장 + 즉시 평가
🔍 harness-review  패턴 분석 + 수정 트리거 판단
🔄 harness-resume  세션 간 컨텍스트 복원

→ 코어를 직접 건드리지 않음. 개선 판단 근거 축적.

3 메타 하네스 스킬별 역할
📊 harness-analysis
세션 시작·종료
코어 스킬의 REJECT율과 토큰 소비를 세션 단위로 측정. JSONL에서 VERDICT 패턴을 감지해 code-reviewer / security-guard의 거부율을 계산.
→ report.json (reject_rates)
📝 harness-log
세션 종료
세션 일지와 블루프린트 스냅샷 저장. 직전 분석 리포트의 reject_rates를 비교해 regression_signal 생성. 다음 에이전트를 위한 핸드오프 프롬프트 작성.
→ logs/ + blueprints/ (regression_signal)
🔄 harness-resume
세션 시작
최근 일지와 블루프린트를 읽어 이전 세션 컨텍스트를 복원. 핸드오프 프롬프트와 다음 할 일을 브리핑해 새 에이전트가 빠르게 이어받을 수 있도록 함.
→ 컨텍스트 브리핑
📚 harness-reference
아티클 읽을 때
URL에서 아티클을 읽고 요약·태그·연결 스킬을 제안. 연결 스킬이 있으면 Golden Rule 기준으로 즉시 평가. 평가 결과를 evaluations/에 누적.
→ references/ + evaluations/
🔍 harness-review
온디맨드
누적된 평가 이력에서 서로 다른 아티클 관점으로 3회 이상 반복되는 갭을 추출. 코어 수정 트리거 기준과 현재 상태를 대조해 사람 판단을 일관되게 지원.
→ 수정 트리거 판단
4 코어 수정 트리거 — 수작업 판단 기준

자동화 아님. 메타 하네스가 신호를 만들고, 사람이 판단해서 코어를 수정.

📊 reject_rate 이탈 → 5회 실측 후 평균 대비 2배↑ → code-reviewer / security-guard 기준 방향 검토 (강화인지 완화인지 판단 필요)
🔍 동일 갭 반복 → 서로 다른 아티클 3종 이상 → 다음 세션 첫 작업으로 해당 스킬 수정 harness-log "내일 바로 할 일"에 기록
📝 needs-work 누적 → 같은 스킬에 2개↑ 아티클 needs-work → harness-review 우선 실행 후 판단 (즉시 수정 아님)
