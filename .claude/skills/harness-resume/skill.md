# Harness Resume — 이전 세션 컨텍스트 복원

새 세션 시작 시 최근 개발 일지와 블루프린트를 읽어 현재 상태를 파악합니다.

## 실행 절차

### 1. 최근 일지 파일 목록 확인

`.harness-lab/logs/` 디렉토리에서 최신 파일 최대 3개를 날짜 역순으로 읽습니다.

파일이 없으면: "아직 저장된 일지가 없습니다. `/harness-log`로 첫 일지를 저장하세요." 출력 후 종료.

### 2. 최신 블루프린트 읽기

`.harness-lab/blueprints/` 디렉토리에서 가장 최근 JSON 파일 1개를 읽습니다.

### 3. 세션 브리핑 출력

아래 형식으로 현재 상태를 정리해 출력합니다:

```
## Harness Resume — {오늘 날짜}

### 최근 작업 (최근 순)
- {날짜}: {작업 요약 한 줄}
- {날짜}: {작업 요약 한 줄}
- {날짜}: {작업 요약 한 줄}

### 현재 아키텍처 상태 ({블루프린트 날짜} 기준)
| 스킬 | 상태 | 버전 |
|------|------|------|
| tdd-guard-claude | DONE | v2.0.0 |
| ... | ... | ... |

커버리지: {current}%
파이프라인: {pipeline}

### 마지막 세션 개선 포인트
{최신 일지의 "개선 포인트" 섹션 내용}

### 이어서 할 일
{최신 일지의 내용을 바탕으로 다음 작업을 추천 1~3개}
```

### 4. 준비 완료 선언

브리핑 출력 후: "컨텍스트 복원 완료. 이어서 작업하시겠습니까?" 로 마무리합니다.
