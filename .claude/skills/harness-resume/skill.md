# Harness Resume — 이전 세션 컨텍스트 복원

새 세션 시작 시 최근 개발 일지와 블루프린트를 읽어 현재 상태를 파악합니다.
Marketplace의 Harness Lab 페이지(`HarnessLabPage.jsx`)의 "오늘 인계" 카드와 동일한 컨텍스트를 CLI 환경에서 제공합니다.

## 실행 절차

### 1. 최근 일지 파일 목록 확인

**우선 로컬 → 없으면 원격 API 조회** 순으로 데이터를 가져옵니다.

**로컬:** `.harness-lab/logs/` 디렉토리에서 최신 파일 최대 3개를 날짜 역순으로 읽습니다.

**원격 폴백:** 로컬에 파일이 없으면 원격 API를 조회합니다:
```bash
curl {서버URL}/api/harness/logs
```
서버 URL은 환경변수 `VITE_API_URL` 또는 기본값 `https://skill-marketplace-umzq.onrender.com` 을 사용합니다.

둘 다 없으면: "아직 저장된 일지가 없습니다. `/harness-log`로 첫 일지를 저장하세요." 출력 후 종료.

### 2. 최신 블루프린트 읽기

블루프린트는 스킬별 개선 이력입니다. 최근 변경이 있었던 스킬들을 파악합니다.

**로컬:** `.harness-lab/blueprints/` 디렉토리에서 가장 최근 날짜의 JSON 파일들을 읽습니다.

**원격 폴백:** 로컬에 없으면:
```bash
curl {서버URL}/api/harness/blueprints
```
응답의 `skills` 배열에서 각 스킬의 `latest` entry를 참조합니다.

### 3. 핸드오프 프롬프트 추출

최신 일지에서 `## 다음 프롬프트` 섹션을 추출합니다. 이 섹션은 아래 3개 항목으로 구성됩니다:

- `### 지금 상태`: 현재 브랜치, 어디까지 완료됐는지
- `### 다음 할 일`: 첫 번째로 실행할 작업
- `### 열린 결정`: 아직 결정되지 않은 것

해당 섹션이 없으면 `## 내일 바로 할 일` 섹션을 대신 사용합니다.

### 4. 최근 평가 갭 읽기

`.harness-lab/evaluations/` 디렉토리에서 최근 30일 이내 파일을 읽습니다.
스킬별로 그룹화하여 반복 등장하는 갭만 추출합니다 (2회 이상 = 구조적 문제).
파일이 없으면 이 섹션은 생략합니다.

### 5. 세션 브리핑 출력

아래 형식으로 현재 상태를 정리해 출력합니다:

```
## Harness Resume — {오늘 날짜}

### 최근 작업 (최근 순)
- {날짜}: {작업 요약 한 줄}
- {날짜}: {작업 요약 한 줄}

### 최근 스킬 변경
| 스킬 | 마지막 변경 | 날짜 |
|------|------------|------|
| tdd-guard-claude | ... | 2026-04-10 |

### 반복 평가 갭 (아티클 기반)
| 스킬 | 갭 | 횟수 |
|------|----|------|
| tdd-guard-claude | 검증 자동화 훅 없음 | 2회 |
← 없으면 이 섹션 생략

### 마지막 세션에서 막힌 점
{최신 일지의 "막힌 점" 섹션 내용}

### 이어서 할 일
{최신 일지의 "내일 바로 할 일" 섹션 내용}

### 핸드오프 프롬프트
**지금 상태**: ...
**다음 할 일**: ...
**열린 결정**: ...
```

### 5. 준비 완료 선언

브리핑 출력 후: "컨텍스트 복원 완료. 이어서 작업하시겠습니까?" 로 마무리합니다.
