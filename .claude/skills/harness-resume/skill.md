# Harness Resume — 이전 세션 컨텍스트 복원

새 세션 시작 시 최근 개발 일지와 블루프린트를 읽어 현재 상태를 파악합니다.
Marketplace의 Harness Lab 페이지(`HarnessLabPage.jsx`)의 "오늘 인계" 카드와 동일한 컨텍스트를 CLI 환경에서 제공합니다.

## 실행 절차

### 1. 최근 일지 파일 목록 확인

**우선 로컬 → 없으면 원격 API 조회** 순으로 데이터를 가져옵니다.

**로컬:** `.harness-lab/logs/` 디렉토리에서 최신 파일 최대 3개를 날짜 역순으로 읽습니다.

**원격 폴백:** 로컬에 파일이 없으면 원격 API를 조회합니다:
```bash
curl {서버URL}/api/harness/logs
```
서버 URL은 환경변수 `VITE_API_URL` 또는 기본값 `https://skill-marketplace-umzq.onrender.com` 을 사용합니다.

둘 다 없으면: "아직 저장된 일지가 없습니다. `/harness-log`로 첫 일지를 저장하세요." 출력 후 종료.

### 2. 최신 블루프린트 읽기

**로컬:** `.harness-lab/blueprints/` 디렉토리에서 가장 최근 JSON 파일 1개를 읽습니다.

**원격 폴백:** 로컬에 없으면:
```bash
curl {서버URL}/api/harness/blueprints
```

### 3. 핸드오프 프롬프트 추출

최신 일지에서 `## 다음 프롬프트` 섹션을 추출합니다.
- 이 섹션은 이전 에이전트가 다음 에이전트를 위해 작성한 핸드오프 프롬프트입니다.
- 프론트엔드의 "다음 에이전트 프롬프트 복사" 버튼과 동일한 내용입니다.
- 해당 섹션이 없으면 `## 내일 바로 할 일` 섹션을 대신 사용합니다.

### 4. 오래된 로그 compacting 확인

`.harness-lab/logs/` 의 파일 수를 확인합니다. **7개를 초과**하면 오래된 로그를 압축합니다:

- 최신 5개: 그대로 유지
- 나머지: 각 파일의 `## 작업 요약` 섹션만 추출해 `.harness-lab/logs/archive.md` 에 날짜 역순으로 append 후 원본 삭제

`archive.md` 형식:
```markdown
# 압축 로그 아카이브

## {날짜}
{작업 요약 내용}

## {날짜}
{작업 요약 내용}
```

### 5. 세션 브리핑 출력

아래 형식으로 현재 상태를 정리해 출력합니다:

```
## Harness Resume — {오늘 날짜}

### 최근 작업 (최근 순)
- {날짜}: {작업 요약 한 줄}
- {날짜}: {작업 요약 한 줄}
- {날짜}: {작업 요약 한 줄}

### 최근 변경된 스킬 파일 ({최신 일지 날짜})
- `경로` — 변경 내용
- `경로` — 변경 내용
→ 다음 세션에서 이 파일들을 우선 확인하세요

### 현재 아키텍처 상태 ({블루프린트 날짜} 기준)
| 스킬 | 상태 | 버전 |
|------|------|------|
| tdd-guard-claude | DONE | v2.0.0 |
| ... | ... | ... |

커버리지: {current}%
파이프라인: {pipeline}

### 마지막 세션에서 막힌 점
{최신 일지의 "막힌 점" 섹션 내용}

### 이어서 할 일
{최신 일지의 "내일 바로 할 일" 섹션 내용}

### 핸드오프 프롬프트
{최신 일지의 "다음 프롬프트" 섹션 내용}
```

`## 스킬 변경 파일` 섹션이 없거나 "없음"이면 해당 항목을 브리핑에서 생략합니다.

### 6. 준비 완료 선언

브리핑 출력 후: "컨텍스트 복원 완료. 이어서 작업하시겠습니까?" 로 마무리합니다.
