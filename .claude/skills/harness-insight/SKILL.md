# Harness Insight — 인사이트 검색

저장된 아티클 인사이트들을 문제 맥락에 맞게 증강해서 꺼내줍니다.
인사이트 자체를 그대로 나열하는 게 아니라, 현재 문제에 비춰서 재해석합니다.

---

## 실행 절차

### 1. 인사이트 수집

`.harness-lab/evaluations/` 디렉토리에서 모든 `*.json` 파일을 읽습니다.

파일이 없으면: "저장된 인사이트가 없습니다. `/harness-reference`로 아티클을 저장하면 인사이트가 쌓입니다." 출력 후 종료.

각 파일에서 아래 필드를 추출합니다:
- `article_title`, `article_url`
- `insights` (새 형식) 또는 `gaps` + `suggestions` (구 형식 — 둘 다 지원)
- `skill`, `date`

### 2. 관련도 판단

입력된 문제 설명과 각 인사이트의 관련도를 판단합니다.

판단 기준:
- 인사이트가 문제의 **원인**과 관련되는가
- 인사이트가 문제의 **해결 방향**에 참고가 되는가
- 인사이트가 문제와 **같은 레이어** (코드 통제 / 토큰 밸런스 / 워크플로우)의 이야기인가

관련 없는 인사이트는 포함하지 않습니다. 3개 이상이면 가장 관련성 높은 것만 선별합니다.

### 3. 증강 출력

각 관련 인사이트에 대해 아래 형식으로 출력합니다:

```
## 관련 인사이트 — {입력된 문제 요약}

### {아티클 제목}
출처 스킬: {skill} | {date}

원문 인사이트:
  "{저장된 인사이트 그대로}"

지금 문제에 비춰보면:
  {현재 문제 맥락에서 이 인사이트가 무엇을 시사하는지 — 1~3문장}

---
```

관련 인사이트가 없으면: "저장된 인사이트 중 이 문제와 직접 관련된 것이 없습니다. 문제를 다른 각도로 설명해보시겠어요?" 출력.

### 4. 사용 후 선택지 제시

출력 후 아래 선택지를 제안합니다:
```
이 인사이트들이 도움이 됐나요?
- 도움됐다면: 오늘 일지에 반영하려면 `/harness-log`를 실행하세요
- 새 아티클에서 인사이트를 추가하려면: `/harness-reference {URL}`
```

---

## 주의사항

- 인사이트를 "해야 한다"는 지시로 변환하지 않는다. 참고 관점으로 제시한다.
- 구 형식(`gaps`/`suggestions`)은 `insights`로 취급하되, 판정적 언어는 중립으로 바꿔서 제시한다.
- 문제 해결을 직접 해주는 스킬이 아니다. 인사이트를 꺼내주는 것이 목적이다.
