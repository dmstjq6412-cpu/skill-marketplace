# 커버리지 기준 및 테스트 품질 원칙

## 커버리지 기준 (80%)

| 케이스 | 예시 |
|---|---|
| **Happy path** | 올바른 입력 → 기대 결과 |
| **오류 처리** | DB 오류, 네트워크 실패, 잘못된 입력 |
| **경계값** | 빈 배열, null, undefined, 0 |
| **회귀** | 버그 수정 시 그 버그 재현 케이스 |

- API 엔드포인트: 성공, 리소스 없음(404), DB 오류(500) 케이스 우선
- 컴포넌트: 렌더링, 상호작용, props 분기 우선

---

## 테스트 품질 원칙

- **이름은 한국어로, 동작을 명확히** — `'데이터 로드 후 목록 렌더링'` > `'test1'`
- **beforeEach에서 mock 초기화** — 감지된 프레임워크의 clearMocks API 사용
- **동작을 테스트, 구현 디테일 말고** — 내부 상태보다 반환값·화면 노출 여부·호출 여부
- **픽스처는 상수로 분리** — `const MOCK_XXX = {...}` 파일 상단에 정의

코드 패턴 예시는 `frontend-patterns.md`, `backend-patterns.md` 참고.
(두 파일 모두 Vitest 기준이며, 다른 프레임워크 사용 시 동등한 패턴으로 대체합니다.)
